## 🎉 新功能：知识库集成

### 📋 更新概述

v0.0.4 版本引入了知识库集成功能，允许 AI 助手在回答问题时先查询知识库，获取相关文档片段，然后结合这些信息提供更准确、更有针对性的回答。

### 🚀 主要新功能

#### 1. 知识库集成
- **智能查询**：当用户使用 `@ai` 触发词时，系统会先查询知识库获取相关信息
- **无缝集成**：知识库查询失败不会影响正常的 AI 对话功能
- **信息增强**：查询到的文档片段会自动添加到系统提示词中，提升回答质量

#### 2. 支持 Dify 知识库 API
- **标准接口**：支持 Dify 平台的知识库 API
- **简单配置**：只需配置 API URL 和 API Key 即可使用
- **灵活控制**：可通过配置开关启用或禁用知识库功能

### 🔧 技术实现

#### 新增组件
1. **KnowledgeApiClient**：专门的知识库 API 客户端
2. **配置扩展**：在 ConfigManager 中添加知识库相关配置
3. **消息处理增强**：ChatListener 集成知识库查询流程

#### API 请求格式
```json
POST: /v1/chat-messages
{
    "inputs": {},
    "query": "用户问题",
    "user": "server"
}
```

#### API 响应处理
- 解析响应中的 `answer` 字段
- 将文档片段包装在 `<useful_info>` 标签中
- 自动追加到系统提示词

### ⚙️ 配置说明

#### 新增配置项
```yaml
# 知识库配置
knowledge:
  # 是否启用知识库功能
  enabled: false
  
  # 知识库 API URL (Dify API)
  api_url: "https://api.dify.ai"
  
  # 知识库 API Key (格式: app-...)
  api_key: "your-knowledge-api-key-here"
```

#### 配置步骤
1. 在 `config.yml` 中设置 `knowledge.enabled: true`
2. 配置正确的 `knowledge.api_url`
3. 设置有效的 `knowledge.api_key`（格式：app-...）

### 📝 使用示例

#### 基本使用
```
玩家: @ai 如何建造红石电路？
系统: [查询知识库] → [获取相关文档] → [生成增强回答]
AI: 根据知识库信息，红石电路的基本原理是...
```

#### 工作流程
1. 用户发送 `@ai` 消息
2. 系统提取问题内容
3. 查询知识库获取相关信息
4. 将知识库信息添加到系统提示词
5. 调用 AI API 生成回答
6. 返回增强后的回答

### 🛡️ 错误处理

#### 容错机制
- 知识库查询失败不影响正常 AI 对话
- API Key 未配置时自动跳过知识库查询
- 网络错误时优雅降级到普通 AI 回答

#### 安全特性
- API Key 在日志中自动脱敏显示
- 支持 HTTPS 加密传输
- 请求超时保护
