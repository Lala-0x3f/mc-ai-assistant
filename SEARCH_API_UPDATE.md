# 网络搜索功能重构 - 使用 v1/responses 接口

## 🔄 **重构概述**

我已经完全重构了网络搜索功能，现在使用 OpenAI 的 `v1/responses` 接口，提供更准确和实时的网络搜索结果。

## 🆕 **新的搜索接口**

### API 端点
```
POST /v1/responses
```

### 请求格式
```json
{
    "model": "gpt-4.1-mini",
    "tools": [
        {
            "type": "web_search_preview"
        }
    ],
    "input": "请简短的回答：今天的北京天气适合出门吗?"
}
```

### 响应格式
```json
{
    "output": [
        {
            "type": "web_search_call",
            "action": {
                "type": "search",
                "query": "北京天气 2025年7月22日"
            }
        },
        {
            "type": "message",
            "content": [
                {
                    "type": "output_text",
                    "text": "今天（2025年7月22日），北京的天气晴朗..."
                }
            ]
        }
    ]
}
```

## 🏗️ **架构变更**

### 新增组件

#### 1. **SearchApiClient 类**
- 专门处理 `v1/responses` 接口
- 独立的错误处理和调试功能
- 支持浏览器模拟请求头

#### 2. **SearchResult 类**
- 封装搜索结果
- 包含搜索查询和结果文本
- 便于数据传递和处理

### 更新的组件

#### 1. **ChatListener**
- 使用新的 SearchApiClient
- 更新搜索响应格式
- 显示搜索查询和结果

#### 2. **ConfigManager**
- 新增搜索相关配置方法
- 支持搜索模型和提示词配置

#### 3. **主类和兼容性**
- 初始化 SearchApiClient
- 更新 RedisChatCompatibility

## ⚙️ **配置更新**

### 新的配置结构
```yaml
# 网络搜索配置
search:
  # 搜索模型
  model: "gpt-4.1-mini"
  
  # 搜索提示词前缀 (会添加到用户输入前)
  prompt_prefix: "请简短的回答："
  
  # 搜索工具类型
  tool_type: "web_search_preview"
```

### 配置说明

- **model**: 搜索使用的模型，推荐 `gpt-4.1-mini`
- **prompt_prefix**: 添加到用户输入前的提示词，可自定义
- **tool_type**: 搜索工具类型，固定为 `web_search_preview`
