# MC AI Assistant v0.2.0 发布说明

## 📅 发布信息
- **版本**: v0.2.0
- **发布日期**: 2026-02-03
- **基于版本**: v0.2.0-rc3
- **主要更新**: 统一 /ai 管理指令、工具调用提示与白名单可见性

## 🎯 核心变更：统一 /ai 管理入口

本版本将管理指令统一到 `/ai` 入口，便于维护与权限控制，同时保留旧指令兼容，确保平滑升级。

## 🚀 主要更新

### 1. /ai 统一管理指令

- **统一入口**：`/ai` 集中管理模型、健康检查与白名单等后台操作
- **子命令结构**：
  - `/ai info` 查看插件与模型信息
  - `/ai tools` 查看启用的工具（知识库/图像/后台指令）
  - `/ai model [modelId]` 查看或切换聊天模型
  - `/ai test <all|model|kb>` 健康检查（模型/知识库）
  - `/ai whitelist <list|add|remove|reload>` 管理白名单
- **兼容旧指令**：`/model`、`/aitest`、`/aicmdwl` 继续可用

### 2. 工具调用提示（玩家可见）

- **知识库查询**与**后台指令执行**被调用时，将向请求玩家发送一行提示：
  - `[AI] 已调用知识库查询`
  - `[AI] 已调用后台指令`
- **风格一致**：提示简洁，不展示细节，与图像工具的用户体验一致

### 3. 执行指令的白名单可见性

- `execute_command` 的 tool schema 中会附带**实时白名单摘要**，帮助 AI 做更准确的决策
- 说明字段会展示部分白名单与总数，避免盲猜导致被拒绝

## 🔄 兼容性说明

- ✅ 旧指令保持兼容：`/model`、`/aitest`、`/aicmdwl`
- ✅ 权限节点不变：`mcaiassistant.admin`、`mcaiassistant.test`
- ✅ 现有配置与数据无需迁移

## ⚙️ 配置变更

本版本无新增或修改配置项：
- `config.yml` 无需调整
- `command-whitelist.yml` 继续生效，用于后台指令白名单

## 🧭 使用示例

```
/ai info
/ai tools
/ai model gpt-4o-mini
/ai test all
/ai whitelist list
```

## 🚀 升级步骤

1. **备份插件**：备份现有 jar 与配置文件
2. **更新插件**：替换为 `mc-ai-assistant-0.2.0.jar`
3. **重启服务器**：重启或重载插件
4. **验证功能**：执行 `/ai info`、`/ai tools` 确认功能正常

## 🎉 总结

v0.2.0 重点提升了后台管理的统一性与可视化体验：

1. **统一入口**：/ai 作为唯一管理入口，结构清晰
2. **可感知调用**：工具调用时提示玩家，交互更透明
3. **更少盲区**：执行指令白名单对 AI 可见，降低失败率
