# 🎨 图像生成功能说明

## 📋 功能概述

MC AI Assistant v0.1.0 新增了图像生成功能，允许玩家通过 AI 对话生成参考图像。

## ✨ 主要特性

### 🤖 智能图像生成
- **触发方式**: 通过 `@ai` 命令请求画图
- **自动识别**: AI 自动识别画图请求并生成图像
- **英文转换**: AI 自动将中文描述转换为英文 prompt

### 🔧 技术特性
- **API 集成**: 支持标准图像生成 API
- **异步处理**: 不阻塞游戏进程
- **错误处理**: 完善的错误提示和重试机制
- **固定比例**: 强制 1:1 正方形比例

### 🎮 用户体验
- **实时通知**: Toast 通知显示生成状态
- **多图像支持**: 支持显示 API 返回的所有图像（通常 4 张）
- **美观显示**: 显示中文 alt 描述，悬停显示英文 prompt
- **可点击链接**: 每张图像都有独立的可点击获取按钮
- **详细信息**: 显示生成用时和图像数量
- **服务器集成**: 通过 `/image create` 命令获取图像

## ⚙️ 配置说明

### 配置文件 (config.yml)
```yaml
# 图像生成配置
image_generation:
  # 是否启用图像生成功能
  enabled: false
  
  # 图像生成 API 地址
  api_url: "http://localhost:8000"
```

### 配置参数说明
- **enabled**: 是否启用图像生成功能
- **api_url**: 图像生成服务的 API 地址

## 🔌 API 接口规范

### 请求格式
```http
POST /create
Content-Type: application/json

{
  "prompt": "beautiful sunset over mountains",
  "ar": "1:1"
}
```

### 成功响应
```json
{
  "job_id": "1234567890abcdef",
  "images": [
    "/images/1234567890abcdef/1234567890abcdef_0.jpg",
    "/images/1234567890abcdef/1234567890abcdef_1.jpg",
    "/images/1234567890abcdef/1234567890abcdef_2.jpg",
    "/images/1234567890abcdef/1234567890abcdef_3.jpg"
  ]
}
```

### 错误响应
```json
{
  "detail": "错误描述信息"
}
```

### HTTP 状态码
- **200**: 请求成功
- **429**: 速率限制超出
- **500**: 内部服务器错误
- **504**: 请求超时

## 🛠️ 技术实现

### 核心组件

#### 1. ImageApiClient
- **功能**: 图像生成 API 客户端
- **特性**: 
  - 异步请求处理
  - 完善的错误处理
  - 超时控制（120秒）
  - 调试日志支持

#### 2. 响应处理
- **工具识别**: 自动识别 `create_image` 工具调用
- **文本保留**: 不依赖 XML 标签，保留文本响应清洁
- **异步生成**: 不阻塞主线程的图像生成

#### 3. 用户界面
- **Toast 通知**: 生成开始和状态提示
- **多图像显示**: 支持显示多张生成的图像
- **美观文本**: 使用中文 alt 描述替代英文 prompt
- **可点击组件**: 使用 Adventure API 创建可点击链接
- **悬停提示**: 显示完整命令信息和原始 prompt
- **详细信息**: 每个图像链接都有独立的悬停提示

### 工作流程

1. **请求识别**: 玩家发送包含画图请求的 `@ai` 命令
2. **AI 处理**: AI 识别请求并发起 `create_image` 工具调用
3. **参数解析**: 插件解析工具参数并提取英文 prompt
4. **异步生成**: 调用图像生成 API 进行异步处理
5. **状态通知**: 显示生成开始的 Toast 通知
6. **结果处理**: 生成完成后显示可点击的获取链接
7. **图像获取**: 玩家点击链接执行 `/image create` 命令

## ⚠️ 注意事项

### 1. 系统要求
- **Java 21+**: 需要 Java 21 或更高版本
- **PaperMC 1.21.4**: 需要 PaperMC 1.21.4 服务器
- **图像生成服务**: 需要部署兼容的图像生成 API 服务

### 2. 性能考虑
- **异步处理**: 图像生成不会阻塞游戏
- **超时控制**: 120秒超时防止长时间等待
- **资源管理**: 自动清理 HTTP 连接

### 3. 安全注意
- **输入验证**: 自动验证和清理 prompt 内容
- **速率限制**: 依赖现有的速率限制机制
- **错误处理**: 不暴露敏感的系统信息

### 4. 兼容性
- **RedisChat**: 完全兼容跨服务器聊天
- **权限系统**: 使用现有的权限节点
- **配置热重载**: 支持配置文件热重载

## 🔧 故障排除

### 常见问题

#### 1. 图像生成失败
- **检查 API 地址**: 确认 `api_url` 配置正确
- **检查服务状态**: 确认图像生成服务正常运行
- **查看日志**: 启用调试模式查看详细错误信息

#### 2. 响应超时
- **网络连接**: 检查服务器到 API 服务的网络连接
- **服务性能**: 确认图像生成服务响应时间正常
- **调整超时**: 如需要可以修改超时时间

#### 3. 权限问题
- **用户权限**: 确认玩家有 `mcaiassistant.use` 权限
- **功能启用**: 确认 `image_generation.enabled` 为 true

### 错误代码说明
- **429**: 请求过于频繁，请稍后再试
- **500**: 服务器内部错误，检查 API 服务
- **504**: 请求超时，检查网络连接

## 📈 性能统计

### 典型性能指标
- **API 调用延迟**: 通常 2-5 秒
- **图像生成时间**: 根据复杂度 3-30 秒
- **内存使用**: 每次请求约 1-2MB
- **网络带宽**: 请求约 1KB，响应约 2-5KB
