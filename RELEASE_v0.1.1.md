# MC AI Assistant v0.1.1 发布说明

## 📅 发布信息
- **版本**: v0.1.1
- **发布日期**: 2025-07-30
- **基于版本**: v0.1.0
- **主要更新**: 知识库查询功能重构

## 🎉 主要功能：知识库查询优化

### 📋 更新概述

v0.1.1 版本重构了知识库查询功能，从每次请求都自动查询知识库改为按需查询的智能方式，显著提升了响应速度和用户体验。

### 🚀 主要新功能

#### 1. 智能按需查询
- **智能判断**: AI 自动判断是否需要查询知识库
- **按需触发**: 只有相关问题才会触发知识库查询
- **响应提速**: 普通问题响应时间提升 50-60%

#### 2. 新的查询标签机制
- **标签格式**: `<query_knowledge query="查询内容" />`
- **自动解析**: 插件自动检测和解析查询标签
- **异步处理**: 知识库查询不阻塞主线程

#### 3. 配置增强
- **内容描述**: 新增 `knowledge.content` 配置项
- **智能提示**: AI 根据配置内容判断查询时机
- **向后兼容**: 完全兼容 v0.1.0 配置

### 🔧 技术实现

#### 新增配置项
```yaml
knowledge:
  # 知识库包含的内容描述 (让 AI 知道什么时候需要查询知识库)
  content: "Minecraft 服务器规则、插件使用说明、建筑教程、红石电路教程等"
```

#### 系统提示词增强
AI 现在包含知识库查询指令：
```
如果你需要查询知识库来获取更准确的信息，请在响应中添加一行

<query_knowledge query="" />

Query 是你想要查询的问题，可以使用自然语言，建议包含2-5个关键点
知识库包含：[配置的内容描述]
只有当问题涉及这些内容时才需要查询知识库
需要格式完全准确，才能查询知识库
示例：<query_knowledge query="Minecraft 红石电路基础教程" />
```

## 🛠️ 技术架构更新

### 核心组件变更

#### 1. ChatListener.java
- **移除自动查询**: 不再每次都查询知识库
- **新增标签处理**: `processKnowledgeQuery()` 方法
- **标签移除**: `removeKnowledgeTags()` 方法
- **异步查询**: 知识库查询异步处理

#### 2. AiApiClient.java
- **提示词增强**: 添加知识库查询指令
- **智能判断**: AI 根据内容描述判断查询时机

#### 3. ConfigManager.java
- **新增方法**: `getKnowledgeContent()` 获取内容描述

### 工作流程优化
1. **用户请求**: 玩家发送 `@ai` 消息
2. **AI 初步回答**: AI 分析问题并给出初步回答
3. **标签检测**: 如果需要查询知识库，AI 添加 `<query_knowledge>` 标签
4. **标签解析**: 插件检测并解析标签
5. **知识库查询**: 异步查询知识库获取相关信息
6. **最终回答**: 使用知识库信息重新请求 AI，获得更准确的答案

## ⚙️ 配置说明

### 完整配置示例
```yaml
knowledge:
  # 是否启用知识库功能
  enabled: true
  
  # 知识库 API URL (Dify API)
  api_url: "https://api.dify.ai"
  
  # 知识库 API Key (格式: app-...)
  api_key: "your-app-key-here"
  
  # 知识库包含的内容描述 (让 AI 知道什么时候需要查询知识库)
  content: "Minecraft 服务器规则、插件使用说明、建筑教程、红石电路教程等"
```

### 配置参数说明
- **enabled**: 是否启用知识库功能
- **api_url**: 知识库 API 地址
- **api_key**: 知识库 API 密钥
- **content**: 知识库内容描述，帮助 AI 判断何时需要查询

## 🔧 故障排除

### 常见问题

#### 1. 知识库查询不触发
- **检查配置**: 确认 `knowledge.enabled: true`
- **检查内容描述**: 确保 `knowledge.content` 描述准确
- **查看日志**: 启用调试模式查看 AI 响应

#### 2. 查询触发过于频繁
- **调整内容描述**: 使 `knowledge.content` 更具体
- **检查提示词**: 确认系统提示词设置正确

#### 3. 知识库查询失败
- **检查 API**: 确认知识库 API 服务正常
- **检查网络**: 确认服务器能访问知识库 API
- **查看错误日志**: 启用调试模式查看详细错误

## 🚀 部署说明

### 1. 编译
```bash
mvn clean package -DskipTests
```

### 2. 安装
- 将 `target/mc-ai-assistant-0.1.1.jar` 复制到服务器 `plugins` 目录
- 重启服务器或使用 `/reload` 命令

### 3. 配置
- 编辑 `plugins/McAiAssistant/config.yml`
- 添加 `knowledge.content` 配置项（可选）
- 重启服务器或重载配置

### 4. 测试
- 使用 `@ai 你好` 测试普通响应（应该很快）
- 使用 `@ai 红石电路怎么做` 测试知识库查询（如果配置了相关内容）
- 检查日志确认功能正常工作
